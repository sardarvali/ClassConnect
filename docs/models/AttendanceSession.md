## 🎯 What This File Does
The `AttendanceRecord` data class represents a single attendance session for a class on a specific date. Teachers create sessions with a QR code; students scan to mark themselves present. The record tracks who is present and absent. Stored at `/classes/{classId}/attendance/{date}` where `date` is an ISO string (e.g., "2026-03-06").

## 📝 Full Annotated Source Code

```kotlin
package com.syed.classconnect.data.model
// Package: data model layer.

import com.google.firebase.Timestamp
// Timestamp: Firebase's time type for expiry and creation times.

data class AttendanceRecord(
// AttendanceRecord: represents one attendance session for one date.
// Stored at: /classes/{classId}/attendance/{date}
    val date: String = "",
    // date: ISO date string like "2026-03-06". Used as the Firestore document ID.
    // Generated by DateUtils.todayIsoString().
    val teacherId: String = "",
    // teacherId: UID of the teacher who created this session.
    val qrToken: String = "",
    // qrToken: A UUID.randomUUID() token embedded in the QR code.
    // Students scan the QR → app validates this token against Firestore.
    val qrExpiresAt: Timestamp = Timestamp.now(),
    // qrExpiresAt: When the QR code becomes invalid.
    // Set to current time + QR_EXPIRY_MINUTES (5 minutes) during session creation.
    val present: List<String> = emptyList(),
    // present: List of student UIDs who successfully scanned.
    // Updated via FieldValue.arrayUnion(studentId) — atomic append.
    val absent: List<String> = emptyList(),
    // absent: List of student UIDs who did NOT scan.
    // Calculated when teacher ends the session: allStudentIds - present.
    val sessionCreatedAt: Timestamp = Timestamp.now()
    // sessionCreatedAt: Server timestamp of when the session was created.
)
```

---

## ⚠️ Important Notes
- The document ID is the ISO date string, so there can only be ONE session per class per day
- QR token validation checks both the token value AND expiry time
- `present` list is updated atomically using `FieldValue.arrayUnion` to prevent race conditions
- `absent` list is only set when the teacher explicitly ends the session
